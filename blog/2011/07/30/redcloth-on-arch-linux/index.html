RedCloth on Arch Linux

Jul 30th, 2011

I’ve been playing around with Octopress to manage this github site. The setup is pretty easy, but I came across some trouble with the RedCloth installation. The compile fails because in gcc 4.6 (which is what is installed on my Arch Linux system), warnings are treated as errors, so the compile process aborts.

I found out that using an older version of gcc will fix this, so I installed gcc 4.5 from the AUR:

Note that I use packer as my AUR helper. You might be using a different helper (or even installing manually, which is up to you, I suppose).

It took about an hour and a half on my dual core system to compile gcc. If you’re lucky and have one of those fancy new processors, it should not take as long.

Once it has installed, you have to set gcc 4.5 as the default compiler. To do this, simply rename /usr/bin/gcc (which is gcc-4.6) to something else (e.g. /usr/bin/gcc-4.6).

Then rename gcc-4.5 to gcc.

Then try installing the RedCloth gem again.
